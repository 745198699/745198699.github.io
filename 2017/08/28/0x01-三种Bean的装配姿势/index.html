0x01 三种Bean的装配姿势

发表于 2017-08-28 | 分类于 Java

Bean，容器，由Spring负责创建，通常单例。

之前了解了DI依赖注入，注入到构造器、setter或者接口方法中的实例，就是Spring生成的Bean。 下面具体说说三种Bean的装配姿势

  1. XML装配
  2. Java代码装配
  3. 自动装配

注意区分装配方式和注入方式，注入方式主要是区分考虑了注入点，是注入的构造器还是setter等，而装配方式则是如果实现构造器、setter的注入

XML装配

由XML文件负责管理bean的生成和注入以及注入的渠道，如通过构造器注入或者通过setter注入

构造器注入

给PianoPlayer定义一个构造器：

然后定义一个xml，用来生成piano的bean，并注入到PianoPlayer的构造器中

  1. bean的id是唯一标识，class则是指定这个bean的类型，如果就指定了这两项，没有指定其他的参数，则调用默认构造器。所以，这里就生成了一个Piano的实例，id是piano
  2. 构造pianoPlayer，在Java代码中已经为其写好了构造器，但是入到传入三个参数，这个注入的操作就是在xml中指定的
  3. 为Player指定了id和class后，添加了三个<constructor-arg>子标签。如果是String/int等类型的，可以直接通过value设定。然后，之前构造好的bean则是通过ref指定bean的id，如piano
  4. 如果是list类型的，则需要<list>标签来实现，类似还有<set>标签。list内部可以设置<value>...</value>和<ref bean="id">来指定是普通数据还是bean的引用。

XML装配是之前经常用的，现在已经流行使用Java的注解进行装配了，不需要写繁琐的XML文件，而是使用一个普通的Java文件替代

接下来编写Java的配置文件：关键的注解有@Configuration和@Bean

  1. @Configuration表示这个类是个Bean的配置类，类似于之前XML的作用，用于生成Bean，并完成注入
  2. 在配置文件中，我们可以自己写方法生成返回类的实例，并添加@Bean，告诉Spring这里生成的实例装配到Bean中

自动装配

所谓自动装配就是不需要配置文件来管理Bean的生成和注入，关键注解：@Configuration、@ComponentScan、@Autowired

ps. 但是仍需要Java配置文件或XML配置文件，只不过无需复杂的bean的生成和注入的代码

  1. @Component会告知Spring，该类要装载成Bean
  2. @Autowired修饰的方法表明：该方法传入的参数是Spring的Bean，自己去找吧
  3. 不论是构造器还是setter还是普通的方法，如果传入的参数是被@Component修饰的，就会自动装配
  4. 自动装配需要通过配置文件开启，可以是Java也可以是XML

解决方案4 自定义注解

万一自定义的@Qualifier也有重复怎么办，想到可以用多个Qualifier来进一步区分。但是不支持多个@Qualifier

虽然Java8支持重复的注解，但是注解定义的时候必须带有@Repeatable才可以，而@Qualifier并不支持
