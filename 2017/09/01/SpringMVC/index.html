SpringMVC

发表于 2017-09-01 | 分类于 Java

在Spring框架的基础上采用了MVC的思想

  • Model（模型）是应用程序中用于处理应用程序数据逻辑的部分。通常模型对象负责在数据库中存取数据。
  • View（视图）是应用程序中处理数据显示的部分。通常视图是依据模型数据创建的。
  • Controller（控制器）是应用程序中处理用户交互的部分。通常控制器负责从视图读取数据，控制用户输入，并向模型发送数据。

体系架构

在整个框架中，DispatcherServlet处于核心地位，负责协调和组织不同组件响应请求并返回。

大体的流程：

  1. 用户发起了HTTP的请求，DispatcherServlet根据请求的URL指定对应的controller
  2. 相应的controller会返回ModelAndView
  3. DispatcherServlet找到对应的View，传递对应Model，生成最后的界面返回给用户

DispatcherServlet

web.xml的配置文件

  1. <servlet>标签申明了DispatcherServlet，指定了name，会默认自动加载/WEB-INF/${NAME}-servlet.xml的配置文件
  2. <servlet-mapping>指定该DispatcherServlet会处理所有以.html结尾的请求，即所有的html请求都会被DispatcherServlet拦截并处理
  3. 具体的servlet配置文件中，会将所有的ModelAndView以及String解析为具体的页面。后面会具体介绍

Controller

关键点：@Controller @RequestMapping

  1. 指定该class是请求的控制器Controller
  2. class的@RequestMapping指定了URL的根路径（前缀），具体方法的@RequestMapping则是指定了具体的URL请求
  3. 方法的返回值可以直接是个String，视图解析器会解析到对应的jsp。如loginPage()处理后请求后，最后返回"login"，对应的是login.jsp
  4. 方法返回类型是ModelAndView。第一个参数也是指定对应的View，后面则是对应的ModelName和具体是数据。如直接返回了new ModelAndView("main")则是直接跳转到了main.jsp，或是return new ModelAndView("login", "error", "用户名或密码错误。"，回到login.jsp，并附加数据，name是error，数据是"用户名或密码错误。"
