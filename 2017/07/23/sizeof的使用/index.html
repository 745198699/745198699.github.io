sizeof的使用

发表于 2017-07-23 | 分类于 C/C++

基本使用

测试环境 Mac OS X 10.12

首先，sizeof并不是方法，而是keyword

sizeof(char)->1

sizeof(short)->2

sizeof(int)->4

sizeof(float)->4

sizeof(double)->8

sizeof(char a[4])->16

sizeof(char *a)->64

数组的长度就是sizeof(元素)*数组长度
指针的长度则是地址长度

struct内存对齐

sizeof(MyStruct)的值并不是sizeof(char)+sizeof(int)+sizeof(double)

因为内存中的数据都是对齐的，不能一个struct中，第1个字节分配给char c，第2到第5分配给int i… （计算机也是有强迫症滴）默认情况下，是以“当前”最大的那个元素的长度对齐。整个Struct的地址必须是最大字节的整数倍

有看到资料说填充的数据是随机的…全局和静态应该回自动初始化0，但是局部变量就随机了

会输出“not equal”

在分配第一个字节给c后，再分配给i空间时，需要对齐，也就是说i在struct中的偏移必须是sizeof(int)的整数倍。所以，会自动填充3个字节。这时候已经分配了1+3+4个字节了。

到给j分配空间了，前面已经分配了8个字节，sizeof(double)正好也是8，所以可以不用填充，直接分配。

最终MyStruct的size是1+3+4+8=16

现在调整一下struct的成员的顺序：

首先分配了4字节给i，然后分配j空间是，会填充4字节，保证j的8字节对齐。
最后分配c时，当前对齐标准是double，所以会分配一个double的空间。最终分配的结果是i(4+4) + j(8) + c(1+7) = 24

program参数

之前提到了会使用当前最大长度作为对齐标准，但是也可以设置自定义的对齐长度：#pragma pack(4)

那上面那个24的例子，

4字节对齐，i分配4字节，j分配8字节，c分配1+3字节，一共16个字节

pack的大小不能随意设置，Expected #pragma pack parameter to be ‘1’, ‘2’, ‘4’, ‘8’, or ‘16’

static成员变量

static成员变量是不会统计到struct的size里面，因为他存储的区域不一样，在静态区

这里，由于j是static的，所以sizeof只考虑i和c，也就是4+(1+3)=8

tips

值得注意的是，如果struct里面什么都没有

sizeof(MyStruct)的值是1

其次class和struct的计算方式一样。毕竟class和struct很像，互相都可以继承。如果子类是struct，则是public继承；如果子类是class，则是private继承。

那就补充一下三种继承方式：

  1. public继承不改变基类成员的访问权限
  2. private继承使得基类所有成员在子类中的访问权限变为private
  3. protected继承将基类中public成员变为子类的protected成员，其它成员的访问 权限不变。
